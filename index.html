Introduction


Video games became wildly popular with arcade machines, but as technology progressed, home consoles were created, too. Before long, they had become a major factor in the electronic entertainment market. Since the 1970s, video games have become more and more popular, and the global video game market is expected to have a value of over 100 billion USD by 2019. I myself am a big fan of video games, but I also love to learn about their history and how the video game industry operates.

Because video games have become such a huge market, marketing is critical to appeal to the right audience and to understand trends in people's interests in certain types of games. Because sales data is made public, not only can I analyze which games and systems have been most popular, but I can also notice trends into which genres will have a higher appeal in the future or into which companies might not be making as many popular games as they used to. If I were working for a video game company, I could use these data science techniques to help my company understand where the industry is headed and how we might be able to profit off of that. Of course, if that were my full-time job, I would also look at social media data to better grasp what people would like in games nowadays. However, for this project, I am only going to be looking at video game sales data.
This data is collected from the Kaggle dataset at https://www.kaggle.com/gregorut/videogamesales, where the data is in a csv file, 'vgsales.csv'. On the user's page, he states that he used BeautifulSoup to scrape the data himself from http://www.vgchartz.com, a database for video game hardware and software sales (this project will just be looking at software). The actual webpage for total software sales used for this project is at http://www.vgchartz.com/gamedb/. According to the Kaggle user's page, this dataset is current as of 10/26/16, so any games released since then, including Nintendo Switch games, are not covered. Additionally, the user states that this dataset contains all listed games that have sold at least 100,000 copies, of which there are 16,598.
Before I begin, I must import any important libraries. Pandas is the main framework for storing and manipulating data in Python. Numpy is a strong mathematical tool for scientific operations in Python. Matplotlib is a library used to make plots in Python to help visualize your data. Sklearn is a machine learning tool capable of power regression analysis.

Tidying the Data


Thankfully the person who scraped the data from the website did a great job to make sure that this data is already quite tidy and readily available for analysis. However, I still want to perform my own check to tidy the data to a form that I prefer. First, I want to create individual lists of all platforms, genres, and years to get a better understanding of what data I have and what needs to be fixed. For better readability, I will also sort these lists.

To access each row of a Pandas DataFrame, you the iterrows() function, which returns an iterrable of each row. Each row is a tuple of the index and the series containg the rest of the information, which you can access with row[1]. A Pandas Series is an array-like data structure that has specific values for each column in the DataFrame. You can see the following code on how to access data from a DataFrame. The append() function is a list function to place the given value at the end of a list (i.e., if you're list = [5,2], and you call append(-4), you get [5,2,-4]).

It also helps that this period included a large number of highly regarded games, such as Final Fantasy VII and Super Mario 64, which were notorious as being "system sellers". Interest in gaming clearly continues to exponenetially climb until the mid-2000s, where it peaks. This is most likely due to a larger appeal to the "casual gamer" with the Nintendo Wii being a huge success.

Machine Learning and Hypothesis Testing


Machine learning is about taking your data and using it to make predictions for other datasets. I decided the best way I could use machine learning for this dataset is to predict what the best selling genres nowadays are. What I want to do is use machine learning techniques to evaluate the distribution of games for each genre up to 2015, use this to predict the genre distribution for 2016, and then compare the results to the actual distribution given in the report above. This is known as supervised learning, where you use training data to predict future data. I will be using numpy's polyfit, which will return a polynomial of best fit that minimizes the least squares error. Furthermore, I will have to perform this for each of the genres separetly and then compare them at the end. The followng code will take each genre and give it a hash where the keys are the years and the values are the percentage of games are that genre for the given year.

def create_fit(deg):

So a degree of 2 is needed to give the best prediction. This means that the data is best represented by a quadratic graph. How good are the predictions? According to the above data, the sum of the root square error = sqrt(.018) = .13. Keep in mind that the data themselves are percentage values as a decimal, so they are all between 0 and 1. To see how well these correlate, I will perform a hypothesis test on the two data sets.

When you perform hypothesis testing, you let the null hypothesis be that there is no relationship between your two datasets, while the alternative hypothesis be that there is a realtionship. One way to test for this is to use the correlation $R^2$ score, known as the coefficient of determination, which is always between 0 and 1. A value of 0 indicates that there is absolutely no relationship, while a value of 1 indiciates an exact one to one correspondence. The closer the value is to 1, the better the relationship. I can use sklearn to identify what my $R^2$ value is:
In [34]:
